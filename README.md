# Proxy GUI Client MVP

Proxy GUI Client 是一个 Windows 桌面端代理连接管理器 MVP。程序本身不实现底层代理协议，而是调用外部代理核心，负责节点管理、订阅解析、外部配置导入、配置生成、核心进程管理、日志展示、Windows 系统代理开关和延迟测试。

## 当前功能

- PySide6 桌面 GUI。
- 手动添加、编辑、删除节点。
- 订阅拉取和解析：`vmess://`、`vless://`、`trojan://`、`ss://`，兼容 Base64 订阅。
- Clash / Mihomo YAML 与 sing-box JSON 配置导入。
- 导入预览、节点去重、导入策略选择。
- 核心适配器抽象：
  - `sing-box`：可用。
  - `Mihomo`：适配器占位，暂未完整实现。
  - `Xray`：适配器占位，暂未完整实现。
- 自动生成 sing-box JSON 配置。
- 启动/停止代理核心，实时读取 stdout/stderr。
- TCP connect 延迟测试。
- URL 代理延迟测试：自动分配临时 HTTP/SOCKS 端口，临时启动核心，通过本地 HTTP 代理访问测试 URL。
- URL 延迟失败诊断：显示临时核心 stdout/stderr 摘要。
- Windows 系统代理开启、关闭、恢复。
- pytest 测试和一键开发者检查脚本。

## 运行

### 一键运行

方法一：双击运行：

```text
run.bat
```

方法二：PowerShell 运行：

```powershell
.\scripts\run.ps1
```

方法三：运行前检查，通过后再启动 GUI：

```powershell
.\scripts\run.ps1 -Check
```

方法四：跳过依赖安装快速启动：

```powershell
.\scripts\run.ps1 -NoInstall
```

也可以组合使用：

```powershell
.\scripts\run.ps1 -Check -NoInstall
```

开发者检查脚本：

```text
scripts\dev_check.bat
```

如果 PowerShell 提示脚本无法运行，可以执行：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

### 手动运行

```powershell
cd C:\Users\Admin-1\Documents\hysteria
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python -m proxy_gui_client.main
```

也支持：

```powershell
python proxy_gui_client\main.py
```

## Windows 打包与便携模式

项目提供 PyInstaller 打包脚本：

```bat
scripts\build.bat
```

脚本会自动创建 `.venv`、安装依赖并执行 one-folder 打包。输出位置：

```text
dist\ProxyGUI\ProxyGUI.exe
```

打包时会包含：

- PySide6 运行依赖。
- `proxy_gui_client/data` 默认数据。
- `cores` 目录中的 sing-box / Mihomo / Xray 核心。
- `version.txt`。

打包后的程序使用便携目录：

- `dist\ProxyGUI\data`：保存 `settings.json`、`nodes.json`、生成配置等可写数据。
- `dist\ProxyGUI\cores`：保存代理核心。
- `dist\ProxyGUI\logs`：保存日志。

开发态会继续使用项目内 `proxy_gui_client/data` 和 `proxy_gui_client/logs`。打包态会自动以 `ProxyGUI.exe` 所在目录作为 `BASE_DIR`，因此复制整个 `dist\ProxyGUI` 文件夹到其他位置也可以运行。

## 更新检查

程序包含轻量自更新检查结构：

- 本地版本从 `version.txt` 读取。
- 设置中可以启用“启动时检查更新”，并填写更新检查 URL。
- URL 可以返回 GitHub release JSON（包含 `tag_name`），也可以返回纯文本版本号。
- 如果发现新版本，会提示用户；当前阶段只实现提示，不会自动下载和替换程序。

## sing-box

1. 下载 Windows 版 `sing-box`。
2. 解压后找到 `sing-box.exe`。
3. 启动本程序，点击“设置”。
4. 在核心 exe 路径中选择实际路径，例如 `C:\Tools\sing-box\sing-box.exe`。

正式代理默认监听：

- HTTP：`127.0.0.1:7890`
- SOCKS5：`127.0.0.1:7891`

## 订阅组

点击“设置”可以管理多个订阅组。每个订阅组包含：

- 启用状态。
- 订阅名称。
- 订阅链接。

点击“添加订阅组”可以新增一组订阅；选中表格行后点击“删除订阅组”可以删除。刷新订阅时，程序会依次拉取所有已启用的订阅组：

- 单个订阅组失败不会影响其他订阅组。
- 成功刷新的订阅组会替换该组之前导入的旧节点。
- 订阅节点会保存来源，来源显示为 `subscription`，`source_file` 保存订阅组名称。
- 旧版本 settings.json 中的单个 `subscription_url` 会自动迁移为“默认订阅”组。

## 配置导入

点击主窗口“导入配置”可以导入：

- Clash / Mihomo YAML：`.yaml`、`.yml`
- sing-box JSON：`.json`

当前支持导入的协议：

- `vmess`
- `vless`
- `trojan`
- `ss` / `shadowsocks`

当前只导入节点，不导入规则、策略组、`proxy-groups`、`rules`、`route`、`selector`、`urltest` 等运行策略。sing-box JSON 中的 `direct`、`block`、`dns` 等非节点 outbound 会被跳过。

### 导入预览与去重

导入前会弹出预览窗口，显示：

- 文件名。
- 解析到的节点总数。
- 新节点数量。
- 与现有节点重复数量。
- 文件内部重复数量。
- warnings / errors 摘要。
- 节点预览表格。第一列可以逐项勾选要导入的节点。

去重不按节点名称判断，而按核心连接参数判断：

- type
- server
- port
- uuid 或 password
- network
- tls
- sni

字段会做大小写和空格规范化，`ss` 和 `shadowsocks` 视为同一类型。名称不参与去重，因为同名节点可能是不同服务器，不同名节点也可能实际相同。

### 导入策略

预览窗口支持逐项勾选。默认勾选规则：

- `new`：默认勾选。
- `duplicate_existing`：默认不勾选。
- `duplicate_inside_file`：默认不勾选。
- `invalid`：不可勾选，也永远不会导入。

预览窗口提供快捷选择按钮：

- 全选：勾选所有可导入项，不包括 `invalid`。
- 全不选：取消所有勾选。
- 只选新增：只勾选 `new`。
- 只选重复：只勾选 `duplicate_existing`。

导入动作包括：

- 导入勾选项：只处理当前勾选的节点；`new` 会追加，`duplicate_existing` 会覆盖已有节点，`duplicate_inside_file` 被勾选时会导入并自动重命名，`invalid` 会跳过。
- 只导入新增：只追加新节点，跳过现有重复和文件内重复。
- 覆盖已有重复：新增节点会追加；与现有节点重复的导入节点会替换已有节点；文件内重复只保留第一次。
- 全部导入并重命名：所有有效节点都追加，不覆盖已有节点；重名会自动加 ` (2)`、` (3)` 后缀。
- 取消：不修改节点列表，不保存 `nodes.json`。

导入成功后，节点会保存来源字段：

- `source`: `config_file`
- `source_file`: 导入文件名
- `imported_at`: ISO 时间字符串

手动节点通常是 `manual`，订阅节点会保留订阅来源。

导入后建议先选中节点点击“测试延迟”，确认 TCP 和 URL 延迟结果，再启动代理。

## 延迟测试

点击“测试延迟”后会依次执行：

- TCP connect 测试：直接连接节点服务器端口，只验证地址和端口是否可达。
- URL 代理测试：临时生成当前节点配置，自动分配独立临时 HTTP/SOCKS 端口，启动临时核心，再通过本地 HTTP 代理访问测试 URL。

URL 延迟测试不会改写正式 `http_port` / `socks_port`，也不会停止当前正在运行的正式代理进程。临时配置写入 `proxy_gui_client/data/tmp/latency_test_*.json`，测试结束后会尽量清理。

默认测试 URL：

```text
https://www.gstatic.com/generate_204
```

可以在“设置”里修改测试 URL 和超时时间。

## 代理模式与规则分流

设置窗口提供“代理模式”：

- 全局模式：所有流量默认走当前代理节点。
- 规则模式：内置基础分流规则，局域网、私有 IP、`.cn`、`.local`、`.lan` 直连，常见广告域名阻断，其余流量走当前代理节点。
- 直连模式：所有流量默认直连，不经过当前代理节点。

当前规则分流先使用内置轻量规则生成 sing-box `route` 配置，不依赖外部规则集。URL 延迟测试会强制使用全局代理模式，避免在直连模式下绕过待测节点。

## 节点搜索、筛选、排序

节点列表上方提供管理工具栏：

- 搜索框：按名称、服务器、类型、来源实时搜索。
- 类型筛选：全部、vmess、vless、trojan、ss / shadowsocks。
- 来源筛选：全部、manual、subscription、config_file、unknown。
- 可用性筛选：全部、TCP 可用、URL 可用、测试失败、未测试。
- 排序：默认顺序、名称、类型、TCP 延迟从低到高、URL 延迟从低到高、来源、导入时间。

筛选和排序只影响当前 GUI 显示，不会改变 `nodes.json` 中节点的真实保存顺序。
节点表格列宽会自动保存到 `settings.json`，下次启动会恢复。搜索、筛选、排序和批量测试范围偏好也会自动保存和恢复。

表格支持多选节点：

- 单击选择一行。
- 按住 `Ctrl` 可选择多个不连续节点。
- 按住 `Shift` 可选择连续范围。

启动代理、编辑、删除等单节点操作仍使用当前焦点行或第一条选中行；批量测试可以使用全部筛选结果，也可以只使用当前选中的节点。

## 批量延迟测试

工具栏提供：

- 测试范围：选择“当前筛选结果”或“当前选中节点”。
- 批量 TCP 测试：对目标范围内的节点测试服务器端口连通性，最大并发 8。
- 批量 URL 测试：对目标范围内的节点执行真实代理 URL 测试，最大并发 2，避免同时启动太多临时核心。
- 停止批量测试：尽量取消尚未开始的任务；正在运行的节点会等待当前测试结束，确保临时核心进程正常清理。
- 清除测试结果：如果已选中节点，只清除选中节点；如果没有选中节点，会询问是否清除当前筛选结果中的所有节点。

批量测试过程中会逐个更新节点的 TCP 延迟、URL 延迟和测试状态。测试结果会保存到 `nodes.json`，下次启动仍可看到最近结果。

## 节点批量管理、导出与备份

节点表格支持多选后执行批量管理：

- 删除选中节点：只删除当前表格中选中的节点。删除前会确认数量，删除时按节点原始 index 倒序处理，避免筛选或排序后删错节点。如果删除的是当前运行节点，程序只在日志区提示，不会自动停止代理进程。
- 导出选中节点：有选中节点时导出选中节点；没有选中节点时会询问是否导出当前筛选结果。
- 备份全部节点：将当前 `nodes.json` 中的全部节点导出为项目 JSON，默认文件名类似 `nodes_backup_YYYYMMDD_HHMMSS.json`。
- 恢复节点备份：选择项目 JSON 备份文件后，可以选择合并或替换。

导出支持三种格式：

- 项目 JSON：当前项目自己的 `ProxyNode` 列表格式，字段最完整，推荐用于备份和恢复。
- sing-box JSON：只导出 `outbounds`，方便复制到 sing-box 配置中继续调整。
- Clash YAML：只导出 `proxies`，方便给 Clash / Mihomo 使用。

导出到 sing-box JSON 或 Clash YAML 时，不支持或字段不完整的节点会被跳过，并在 GUI 日志区显示 warning；不会因为单个坏节点导致整个导出失败，除非没有任何可导出的节点。

恢复节点备份时：

- 合并到当前节点列表：按节点核心连接参数去重，重复节点不会重复添加。
- 替换当前全部节点：清空当前节点列表并使用备份内容替换，执行前会二次确认。

## 批量编辑与清理

选中多个节点后点击“批量编辑”，会打开批量编辑对话框。只有勾选的项目会被修改：

- 修改 `source`：可批量设置为 `manual`、`subscription`、`config_file`、`unknown`。
- 修改 `source_file`：可清空或设置为指定文本。
- 添加名称前缀：例如 `[HK] `。
- 添加名称后缀：例如 ` - backup`。
- 清除延迟测试结果：清空 TCP 延迟、URL 延迟、测试状态和最后测试时间。

点击“清理不可用”会先生成预览，确认后才删除。默认检查当前筛选结果；如果已选中节点，会询问是否只检查选中节点。不可用判断规则：

- `test_status` 包含 `失败` 或 `failed`。
- 或 TCP 延迟和 URL 延迟都为空，并且 `last_tested_at` 不为空。

未测试节点不会被当作不可用节点删除。

点击“清理重复”会按节点核心连接参数分组，不按名称判断重复。去重字段包括 type、server、port、uuid/password、network、tls、sni。预览中每组默认保留第一个节点，删除后续重复节点；确认后才删除。

所有批量编辑和清理操作都会先预览或确认，不会静默删除节点。删除时使用原始节点 index，筛选或排序不会导致删错节点。

## 系统托盘与桌面行为

程序启动后会创建系统托盘图标。托盘 tooltip 会显示当前连接状态，例如未连接、正在连接、已连接节点名或启动失败原因。

托盘图标支持：

- 双击托盘图标：显示或隐藏主窗口。
- 右键菜单“显示主窗口 / 隐藏主窗口”：手动控制窗口显示。
- “启动代理 / 停止代理”：对当前选中节点执行启动或停止。
- “开启系统代理 / 关闭系统代理”：快速切换 Windows 系统代理。
- “测试当前节点延迟”：对当前选中节点执行 TCP 和 URL 延迟测试。
- “设置”：打开设置窗口。
- “退出程序”：真正退出程序，停止核心进程并恢复由程序改动的系统代理。

设置窗口中可以配置：

- 最小化到托盘。
- 关闭窗口时隐藏到托盘。
- 启动时最小化到托盘。
- 开机自启。
- 启动后自动连接上次节点。
- 托盘通知。

如果启用了“关闭窗口时隐藏到托盘”，点击窗口右上角关闭按钮不会退出程序，代理核心也不会被直接杀掉。要彻底退出，请使用托盘菜单“退出程序”。

### 托盘快速切换节点

托盘菜单提供：

- 快速切换节点：显示延迟较低的前 20 个节点，每项包含名称、类型和 TCP/URL 延迟摘要。
- 最近使用节点：保存最近成功启动或切换的 10 个节点，不存在的节点会自动跳过。

点击节点后会保存为当前节点。如果代理正在运行，会停止当前核心、重新生成配置并启动新节点；如果代理未运行，则只切换当前节点并记录最近使用。

### 开机自启与自动连接

开机自启使用当前用户注册表：

```text
HKCU\Software\Microsoft\Windows\CurrentVersion\Run
```

启动命令指向项目根目录的 `run.bat`，不需要管理员权限。路径中有空格时会自动加引号。

启用“启动后自动连接上次节点”后，程序会读取上次成功启动或切换节点的连接身份，在节点列表中找到对应节点并自动启动代理。如果核心路径错误、节点已不存在或配置无效，GUI 日志区会显示失败原因。

## 延迟测试失败排查

URL 延迟测试失败时，GUI 日志区会显示：

- 测试 URL。
- 临时 HTTP 端口。
- 失败原因和错误类别。
- 临时核心 stderr 最后若干行。
- 临时核心 stdout 最后若干行。

日志摘要会自动截断，只保留最后 20 行或最后约 4000 字符。常见失败原因包括核心路径错误、节点参数错误、临时端口被抢占、测试 URL 被网络阻断、节点认证失败、节点本身不可用。

## 系统代理

“开启系统代理”会设置当前 Windows 用户代理：

```text
http=127.0.0.1:7890;https=127.0.0.1:7890;socks=127.0.0.1:7891
```

程序开启系统代理前会记录原始状态。停止代理或退出程序时，可以恢复启动前的系统代理设置。若程序异常退出，下次启动会提示是否按上次记录恢复。

## 常见问题

### sing-box.exe 路径错误

启动时会提示核心路径不存在或不是文件。请在“设置”中重新选择真实的 `sing-box.exe`。

### 端口被占用

正式代理端口被占用时，正式核心可能启动失败。URL 延迟测试使用独立临时端口，一般不受正式代理端口占用影响；如果极小概率遇到临时端口被抢占，重新测试即可。

### 导入后节点不可用

先使用“测试延迟”。TCP 成功但 URL 失败时，通常说明协议认证、TLS、传输参数或远端代理链路有问题。查看 GUI 日志中的临时核心 stderr/stdout 摘要。

### 托盘图标看不到

Windows 可能把新托盘图标收进隐藏区域。请点击任务栏托盘区域的上箭头查看，或在 Windows 托盘设置中允许显示该图标。

### 关闭窗口后程序还在运行

如果启用了“关闭窗口时隐藏到托盘”，关闭按钮只会隐藏主窗口。请从托盘右键菜单选择“退出程序”来真正退出。

### 开机自启没有生效

请检查设置中“开机自启”是否开启，并确认 Windows 没有阻止 `run.bat` 启动。开机自启写入当前用户注册表，不需要管理员权限。

### 自动连接失败

常见原因是上次节点已删除、核心路径错误、端口被占用或节点参数无效。查看 GUI 日志区中的启动失败原因。

### 退出后系统代理未恢复

优先使用托盘菜单“退出程序”。如果程序异常退出，下次启动会检测到上次可能未恢复系统代理，并提示是否恢复。
